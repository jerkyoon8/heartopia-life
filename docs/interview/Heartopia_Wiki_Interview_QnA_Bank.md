## ☕ 1. Java & Spring Boot 기본기

**Q1. Java 17을 사용하셨는데, 이전 버전(Java 8이나 11) 대비 특별한 장점을 체감하신 게 있나요?**
* **A:** "제일 많이 체감한 것은 `Record` 클래스와 `Text Blocks(""" """ )` 지원입니다. 긴 문자열 처리 시 가독성이 좋아졌고, Null 처리(NPE) 시 어느 변수에서 에러가 났는지 정확히 짚어주는 기능이 있어서 디버깅이 편했습니다. 또한 `Stream API` 등 자바 8의 기능을 활용해 로직 최적화(`MapController`의 `enrichPinDetails`)를 진행했습니다."

**Q2. Spring Boot의 핵심 개념인 IoC와 DI가 무엇인지, 본인 프로젝트 어디에 쓰였는지 설명해보세요.**
* **A:** "IoC(제어의 역전)는 객체의 생성을 개발자가 아닌 스프링 컨테이너가 하는 것을 의미하고, DI(의존성 주입)는 생성된 객체를 필요한 곳에 넣어주는 기술입니다. 제 프로젝트에서는 `@RequiredArgsConstructor`를 사용하여 `WikiController` 내부에 `CollectionService`를 생성자 다형성으로 주입받은 것이 대표적인 DI의 활용입니다."

**Q3. Controller와 RestController의 차이는 무엇입니까? 프로젝트에선 어떻게 나누어 쓰셨죠?**
* **A:** "`@Controller`는 주로 화면(HTML/Thymeleaf 템플릿)을 반환할 때 사용하고, `@RestController`는 내부적으로 `@ResponseBody`를 포함하고 있어 화면이 아닌 순수 데이터(JSON)를 반환할 때 씁니다. 이번 프로젝트에서는 `WikiController` 같이 화면 전이를 담당하는 곳은 `@Controller`를 썼고, 실시간 지도 핀 정보를 주거나 검색 자동완성을 주는 API 경로(`@ResponseBody` 사용 영역)는 데이터 통신용으로 분리했습니다."

**Q4. 로그를 남길 때 `System.out.println`을 쓰지 않고 굳이 `Slf4j` 같은 로거를 쓴 이유는 무엇입니까?**
* **A:** "`System.out.print`는 동기식으로 작동되어 서버 트래픽이 몰리면 심각한 성능 저하를 유발합니다. 또한 서버에 남길 때 파일로 저장하거나, 개발 환경에서는 다 보고 운영 환경에서는 `WARN` 이상만 보게 하는 식의 '레벨 조절 기능'이 없기 때문에 반드시 실무형 로거 패키지(`@Slf4j`)를 사용해야 합니다."

## 🗄️ 2. Database (MySQL & MyBatis)

**Q5. MyBatis에서 변수를 넘길 때 `#{}` 와 `${}` 의 차이는 무엇인가요?**
* **A:** "`#{}`는 PreparedStatement 기반으로 파라미터가 바인딩되어서 SQL Injection 해킹 방어에 필수적입니다. 반면 `${}`는 문자열을 그대로 직접 쿼리에 갖다 붙이는 방식이라 해킹에 취약합니다. 제 쿼리(`CollectionMapper.xml`)에서는 값 바인딩에 모두 `#{}`을 사용하여 보안을 챙겼습니다."

**Q6. DB Connection Pool (HikariCP)이 무엇이고 왜 필요합니까?**
* **A:** "클라이언트가 DB에 연결할 때마다 네트워크를 뚫고 커넥션을 맺으려면 시간이 아주 오래 걸립니다. 그래서 어플리케이션을 켤 때 DB 커넥션을 미리 수십 개(Max: 20개) 맺어두고 웅덩이(Pool)에 보관해 뒀다가, 요청이 오면 바로 빌려주고 돌려받는 구조입니다. 이를 통해 쿼리 응답 속도를 획기적으로 개선합니다."

**Q7. 만약 도감 데이터가 갑자기 100만 건으로 늘어났는데 이름 검색(`LIKE '%가물치%'`)이 미친 듯이 느려진다면 어떻게 해결할 건가요?**
* **A:** "전체 테이블을 다 뒤지는 Full-Scan이 일어나서 그렇습니다. 일반적인 조회라면 도감의 '이름(name)' 컬럼에 **Index(인덱스)**를 생성해서 B-Tree 방식의 고속 조회가 가능하도록 DB 튜닝을 진행하겠습니다."

**Q8. JPA(ORM)가 유행인데 왜 굳이 MyBatis를 선택하셨나요?**
* **A:** "간단한 CRUD는 JPA가 편하지만, 도감과 지도처럼 통계나 조건 분기(아이템 타입에 따라 이미지를 다르게 뿌리는 등)가 잦은 구조에서는 복잡한 쿼리를 직관적으로 작성하고 튜닝하기에 직접 SQL을 제어하는 MyBatis가 더 유리하다고 판단했기 때문입니다."

## 🌐 3. Web & Network & Security

**Q9. CSRF 공격이 무엇이며, 프로젝트에서 어떻게 방어했나요?**
* **A:** "CSRF는 해커가 일반 유저를 속여서, 유저의 로그인된 브라우저 권한으로 강제로 특정 요청(서버 자원 삭제/수정 등)을 보내게 만드는 해킹 수법입니다. 이를 막기 위해 저는 스프링 시큐리티의 CSRF 보호를 켰고, 프론트엔드 자바스크립트에서 POST 통신을 할 때 서버가 미리 심어둔 `csrfToken`을 편지 봉투 인장처럼 헤더에 동봉해서 위조된 요청이 아님을 증명하도록 방어했습니다."

**Q10. SSR (Thymeleaf) 방식과 CSR (React, Vue) 방식의 차이가 무엇인가요?**
* **A:** "SSR은 서버에서 완성된 HTML 껍데기를 만들어 브라우저에 내려주는 방식이고, CSR은 텅 빈 화면을 내려준 뒤 브라우저가 자바스크립트로 화면을 직접 그리는 방식입니다. 제 프로젝트는 **'위키 백과사전'** 성격이므로, 구글이나 네이버 검색봇이 정보를 읽어가기 쉬운(SEO 검색 최적화) SSR 방식이 훨씬 유리하다고 판단하여 Thymeleaf를 채택했습니다."

**Q11. HTTP Method(GET, POST, PUT, DELETE)를 `MapController`에서 잘 나누어 쓰셨는데 그 이유가 뭡니까?**
* **A:** "RESTful API 설계 원칙을 지키기 위함입니다. 과거엔 생성/삭제 모두 백엔드에서 GET/POST 두 개로만 대충 뭉뚱그려 쓰기도 했지만, 액션의 목적에 맞게 Method를 세분화해야 프론트엔드 개발자와 소통하기도 편하고, URL(자원) 이름만 봐도 어떤 행위인지 예측할 수 있기 때문입니다."

## 🏗️ 4. Architecture & Design Pattern

**Q12. MVC 패턴을 사용하셨는데, Controller와 Service는 왜 분리해 두나요? 컨트롤러에서 다 하면 안 되나요?**
* **A:** "유지보수와 재사용성 때문입니다. Controller는 브라우저(HTTP)의 요청을 안내하는 '문지기/접수처' 역할만 해야 하고, 핵심 연산 로직을 Service에 ('공장') 모아두어야 나중에 앱(모바일) 버전 서버를 추가로 띄울 때 Service 클래스는 건들지 않고 부품 돌려막기처럼 그대로 재사용할 수 있습니다."

**Q13. DTO 패턴을 쓴 이유를 구체적으로 말씀해 보세요.**
* **A:** "DB와 연결된 객체(Entity)는 무겁고 민감한 데이터(예: 관리자 비밀번호, 수정 날짜 등)가 다 들어있습니다. 화면에서는 도감 이름과 사진 정도만 필요한데 무겁게 다 던질 필요가 없죠. 또한, 프론트 UI 명세가 바뀔 때 DTO가 중간 필터 역할을 해 주어 핵심 Entity가 더러워지는 강결합 현상을 완벽히 차단시켜 줍니다."

## 🛠️ 5. Trouble Shooting & Soft Skills

**Q14. 본인이 작성한 코드가 아닌, 구글링(혹은 AI)으로 가져온 레거시/낯선 코드를 파악할 때 본인만의 노하우가 있습니까?**
* **A:** "저는 항상 '디버그 모드(Break Point)'나 로그를 적극 활용합니다. 낯선 컨트롤러부터 시작해서 데이터가 어디로 흘러가는지(`Controller -> Service -> Mapper`) 브레이크 포인트를 걸고 하나씩 스텝인(Step Into) 하면서, 입력값이 결과값으로 변환되는 로직 스펙을 먼저 노트에 정리한 후 구조를 파악합니다."

**Q15. 이 프로젝트를 진행하면서 개인적으로 가장 크게 성장했다고 느끼는 부분은 무엇입니까?**
* **A:** "어플리케이션(Java) 레벨과 DB(SQL) 레벨을 조화롭게 분산시키는 법을 배웠다는 점입니다. 과거엔 무조건 모든 데이터를 JOIN 쿼리로 짜 맞추려 했지만, 이번 `MapController`의 구현 과정에서 Java Stream을 통해 메모리상에서 객체를 필터링하고 병합하는 것이 DB 병목을 막는 훌륭한 튜닝 기법이 될 수 있다는 걸 깊이 깨달았습니다."
